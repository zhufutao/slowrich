> 版本：v1.0 | 日期：2026-05-17 | 作者：产品经理

## 1. 项目概述

### 1.1 项目背景

A股市场中，大跌抄底是一种常见的投资策略，但投资者往往缺乏量化依据来判断"跌多少抄底、何时卖出"更为合理。慢富项目旨在通过历史数据回测，为投资者提供科学的抄底策略参考，帮助用户找到适合不同市场环境的最优参数组合。

### 1.2 项目目标

- 提供一键式股票行情数据下载与回测分析平台
- 量化评估不同抄底参数组合的收益表现，推荐最优策略
- 可视化展示回测结果，降低投资决策门槛
- 适配PC和手机端，随时随地查看

### 1.3 目标用户

| 角色 | 描述 | 权限 |
|------|------|------|
| 管理员 | 系统运维人员，管理系统和股票池 | 全部功能 + 股票代码管理 + 用户管理 |
| 普通用户 | 个人投资者，使用回测分析功能 | 数据查看 + 回测 + 行情查看 |

### 1.4 成功指标

- 用户可完成从选股 → 下载数据 → 回测 → 查看推荐的完整流程
- 回测结果支持至少10年历史数据，年度统计粒度
- 页面首屏加载 < 3s，回测计算 < 30s（单只股票10年数据）
- PC和手机端均可正常使用

### 2.2 M1 账号登录注册系统

#### 2.2.1 用户故事

- 作为管理员，我希望使用默认账号(admin@666/666666)登录系统，以便管理平台
- 作为普通用户，我希望通过邮箱注册账号，以便使用回测功能
- 作为普通用户，我希望登录后系统记住我的身份，以便无需重复登录

#### 2.2.2 功能规格

**F1.1 用户注册**
- 输入：邮箱、密码（6-20位）、确认密码
- 校验：邮箱格式、密码长度、两次密码一致、邮箱唯一性

**F1.2 用户登录**
- 输入：邮箱、密码
- 校验：邮箱存在、密码正确
- 登录成功返回JWT Token，前端存储至localStorage
- Token有效期7天，过期后需重新登录

**F1.3 用户登出**
- 清除前端Token，跳转至登录页

**F1.4 默认管理员**
- 系统预置管理员账号：`admin@666` / `666666`，role=admin
- 管理员无法被删除，密码可修改

**F1.5 权限控制**
- 管理员：全部功能 + 股票代码CRUD + 用户管理
- 普通用户：数据查看 + 回测 + 行情查看

### 2.4 M3 数据下载引擎

#### 2.4.1 用户故事

- 作为用户，我希望选择股票和时间区间后一键下载行情数据，以便用于回测
- 作为用户，我希望下载中断后可以断点续传，避免重复下载
- 作为用户，我希望下载出错时能看到明确的错误提示，以便排查问题

#### 2.4.2 功能规格

**F3.1 数据源方案**

| 数据源 | 优先级 | 说明 |
|--------|--------|------|
| AKShare | 主数据源 | 免费、无需Token、接口丰富，适合A股日K线数据 |
| Tushare | 备用数据源 | 需Token、数据质量高，作为AKShare不可用时的备选 |

- 主备切换逻辑：AKShare请求失败（网络超时/接口异常）时自动切换Tushare重试
- 格式一致性：两个数据源返回字段统一映射为标准格式（open/high/low/close/pre_close/pct_chg/volume/amt），Tushare字段名差异在后端适配层处理
- 数据源优先级可在下载任务中手动指定，也可自动选择

**F3.2 创建下载任务**
- 选择：股票（下拉选择）、起始日期、结束日期
- 校验：结束日期 ≥ 起始日期、时间区间 ≤ 5年
- 创建后任务状态：pending → running → completed/failed/partial

**F3.3 下载进度**
- 实时展示：已下载天数/总天数、当前进度百分比
- 状态：pending(等待中) / running(下载中) / completed(已完成) / failed(失败) / partial(部分完成)

**F3.4 断点续传**
- 下载中断（网络/服务端错误）后，已下载的数据保留在D1
- 用户可点击"续传"按钮，从断点日期继续下载
- 续传时检查D1中已有数据的最后日期，从下一个交易日开始

**F3.5 错误处理**
- 下载失败时弹窗提示：错误类型 + 错误详情 + 建议操作
- 常见错误码：
  - `DATA_SOURCE_UNAVAILABLE`：数据源不可用，建议切换数据源
  - `RATE_LIMITED`：请求频率过高，请稍后重试
  - `STOCK_CODE_INVALID`：股票代码无效或已退市
  - `NETWORK_ERROR`：网络错误，请检查网络后重试

**F3.6 下载任务列表**
- 展示所有下载任务：股票名称、时间区间、状态、进度、创建时间
- 支持按状态筛选
- 失败/部分完成的任务可点击"续传"或"重试"

#### 2.4.3 页面结构

| 页面 | 路由 | 说明 |
|------|------|------|
| 下载任务页 | /download | 创建任务 + 任务列表 + 进度展示 |

### 2.6 M5 策略回测

#### 2.6.1 用户故事

- 作为用户，我希望选择股票、初始资金和回测区间后一键运行回测，以便评估抄底策略
- 作为用户，我希望回测参数（跌幅阈值、卖出策略）可自定义配置，以便灵活测试不同策略组合
- 作为用户，我希望看到年度统计和推荐策略，以便找到最优参数

#### 2.6.2 功能规格

**F5.1 回测参数配置**

所有参数均支持用户自定义，系统提供默认值：

| 参数 | 默认值 | 自定义范围 | 说明 |
|------|--------|-----------|------|
| 股票 | - | 股票池中选择 | 回测标的 |
| 初始资金 | 100,000 | 10,000 ~ 10,000,000 | 回测起始资金（元） |
| 回测起始日期 | - | 有行情数据的日期范围 | 回测开始时间 |
| 回测结束日期 | - | 有行情数据的日期范围 | 回测结束时间 |
| 跌幅起始值 | 3% | 1% ~ 15%（步长0.5%） | 抄底触发跌幅下限 |
| 跌幅结束值 | 10% | 跌幅起始值 ~ 20%（步长0.5%） | 抄底触发跌幅上限 |
| 跌幅步长 | 0.5% | 0.1% ~ 2% | 每轮跌幅增加量 |
| 卖出策略-次日收盘 | 开启 | 开启/关闭 | 次日收盘卖出 |
| 浮盈卖出起始值 | 5% | 1% ~ 20%（步长0.5%） | 盘中浮盈卖出下限 |
| 浮盈卖出结束值 | 10% | 浮盈卖出起始值 ~ 30%（步长0.5%） | 盘中浮盈卖出上限 |
| 浮盈卖出步长 | 0.5% | 0.1% ~ 2% | 浮盈卖出每档增量 |

**F5.2 回测规则**

每个跌幅阈值（如3%、3.5%、4%...10%）为一轮，每轮内测试所有卖出策略组合：
1. **触发条件**：当日收盘跌幅 ≥ 本轮跌幅阈值时，以收盘价买入（全仓）
2. **卖出策略A - 次日收盘卖出**：次日收盘价卖出
3. **卖出策略B - 盘中浮盈卖出**：从浮盈起始值到结束值，每档测试一次（如5%、5.5%、6%...10%），当盘中涨幅达到目标浮盈时卖出（以当日最高价模拟盘中价），若未达目标则收盘价卖出

**F5.3 统计指标（每个策略组合 × 每年）**

| 指标 | 说明 |
|------|------|
| 触发次数 | 本年度抄底信号触发次数 |
| 年化收益率 | (年末资金 - 年初资金) / 年初资金 × 100% |
| 胜率 | 盈利次数 / 总交易次数 × 100% |
| 盈利次数 | 卖出价 > 买入价的交易次数 |
| 亏损次数 | 卖出价 ≤ 买入价的交易次数 |
| 最大回撤 | 年内资金曲线最大回撤幅度 |
| 当年股市环境备注 | 根据年度涨跌幅自动标注：牛市(年涨幅>20%)、震荡市(-20%~20%)、熊市(年跌幅>20%) |

**F5.4 推荐策略评判标准（量化）**

| 等级 | 名称 | 评判规则 |
|------|------|---------|
| 🥇 黄金策略 | 最优推荐 | 在所有策略组合中，**综合评分最高**的策略。综合评分 = 年化收益率 × 40% + 胜率 × 30% + (1 - 最大回撤) × 30%，取回测区间内所有年份的平均综合评分 |
| 🥈 白银策略 | 稳健推荐 | **胜率 ≥ 50% 且最大回撤 ≤ 20%** 的策略中，年化收益率排名第二的策略 |
| 🥉 青铜策略 | 保守推荐 | **胜率 ≥ 40% 且最大回撤 ≤ 30%** 的策略中，年化收益率排名第三的策略 |

注：若白银/青铜候选不足，从剩余策略中按综合评分递补。

**F5.5 年度浮盈与资金曲线**

- 每年展示：年初资金、年末资金、年度浮盈（元）、年度收益率
- 全回测区间的资金曲线图（折线图）

**F5.6 回测结果展示**

- 回测概览：推荐策略（黄金/白银/青铜）及核心指标
- 年度统计表：所有年份 × 选定策略的统计指标
- 策略对比表：所有跌幅阈值 × 卖出策略的汇总统计
- 资金曲线图：年度资金变化趋势

#### 2.6.3 页面结构

| 页面 | 路由 | 说明 |
|------|------|------|
| 回测配置页 | /backtest | 参数配置表单 |
| 回测结果页 | /backtest/:id | 结果概览 + 年度统计 + 策略对比 + 资金曲线 |

### 2.7 M6 大盘见顶指标

**F6.2 数据抓取策略**

| 项目 | 说明 |
|------|------|
| 更新频率 | 每个交易日收盘后自动抓取一次（15:30触发） |
| 抓取方式 | 后端定时任务，通过Cloudflare Workers Cron Trigger调度 |

| 数据存储 | 抓取成功后存入market_indicators表，同日期数据更新覆盖 |
| 历史数据 | 首次使用时批量抓取近3年历史数据，后续增量更新 |

**F6.3 温度展示**
- 当前温度值 + 温度等级（极冷/冷/适中/热/极热）

- 温度分位说明（当前值在历史数据中的百分位）

**F6.4 扩展预留**
- 接口设计预留多指标扩展（如PE/PB分位、股债利差等）

### 3.3 关键交互流程

**流程1：数据下载与断点续传**
```
选择股票+时间区间 → 点击"开始下载" → 创建任务(pending) → 后端开始下载(running)
  → 进度条实时更新 → 完成(completed)
  → 中途失败(failed/partial) → 弹窗提示错误 → 用户点击"续传" → 从断点继续(running)
```

**流程2：策略回测**
```
选择股票 → 填写初始资金/日期区间 → 配置跌幅/卖出参数(或用默认值)
  → 点击"开始回测" → 后端计算(loading) → 展示结果页
  → 查看推荐策略 → 切换年度/策略对比 → 查看资金曲线
```

## 4. 交互设计要求

### 4.1 响应式设计

- **PC端**（≥1024px）：侧边栏导航 + 内容区，表格完整展示
- **手机端**（<768px）：底部Tab导航，表格横向滚动，表单纵向排列
- **平板端**（768-1023px）：可折叠侧边栏 + 内容区

### 4.2 关键交互说明

- 所有表单提交前需前端校验，非法输入实时提示
- 数据下载进度条每2秒轮询更新
- 回测计算中显示loading动画，预计耗时提示
- 删除操作需二次确认弹窗
- 错误提示使用Toast通知，5秒自动消失；严重错误使用弹窗

## 5. 非功能性需求

### 5.1 性能要求

- 页面首屏加载 < 3s
- API响应时间 < 500ms（回测接口除外，允许30s）
- 回测计算 < 30s（单只股票10年数据）
- 支持同时10个用户在线使用

### 5.2 部署架构

| 组件 | 技术方案 | 说明 |
|------|---------|------|
| 前端 | React + Vite + TailwindCSS | 部署到Cloudflare Pages |
| 后端 | Cloudflare Workers (Hono) | Serverless API |
| 数据库 | Cloudflare D1 (SQLite) | 边缘数据库 |
| 定时任务 | Cloudflare Workers Cron | 市场温度抓取 |

### 5.3 Cloudflare D1 兼容性注意事项

- D1基于SQLite，不支持RIGHT JOIN / FULL OUTER JOIN，需用LEFT JOIN + UNION替代
- D1单次查询返回行数建议 ≤ 1000，大数据量需分页
- D1不支持存储过程和触发器，业务逻辑在Workers层实现
- D1单数据库大小限制10GB，行情数据量大时需考虑分表或归档策略
- D1写入并发有限，批量插入建议使用事务（单次事务 ≤ 100条）

### 5.4 浏览器兼容性

- Chrome 90+、Firefox 90+、Safari 14+、Edge 90+
- 移动端：iOS Safari 14+、Android Chrome 90+
